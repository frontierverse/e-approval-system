# 사내전자결재 시스템 개발 계획서

이 문서는 전체 기능을 한 번에 만들기 위한 문서가 아니다. 작은 작업을 하나씩 끝내면서 사내전자결재 시스템을 완성하기 위한 투두리스트다.

## 진행 원칙

- [ ] 한 번에 하나의 작은 기능만 구현한다.
- [ ] 각 작업은 화면, 데이터, 권한, 검증 기준을 함께 확인한다.
- [ ] 기능을 만들기 전에 임시 목업 데이터로 화면 흐름을 먼저 잡는다.
- [ ] 실제 DB 작업은 도메인 모델을 확정한 뒤 시작한다.
- [ ] 완료한 작업은 이 문서의 체크박스를 갱신한다.

## MVP 범위

첫 번째 동작 버전에서 반드시 필요한 기능이다.

- [x] 로그인한 사용자를 구분할 수 있다.
- [x] 사용자는 결재 문서를 작성할 수 있다.
- [x] 사용자는 결재선을 지정할 수 있다.
- [x] 문서를 결재 요청할 수 있다.
- [x] 결재자는 받은 결재함에서 문서를 확인할 수 있다.
- [x] 결재자는 승인 또는 반려할 수 있다.
- [x] 작성자는 진행 상태를 확인할 수 있다.
- [x] 문서별 결재 이력이 남는다.

첫 번째 버전에 넣지 않아도 되는 기능이다.

- [ ] SSO 연동
- [x] 모바일 앱 제외: 웹서비스로만 제공한다.
- [ ] 메신저 연동
- [ ] 외부 HR 시스템 연동
- [ ] 복잡한 전결/대결 규칙
- [ ] 전자서명 인증서 연동
