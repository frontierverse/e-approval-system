# 문서양식 디자이너 개발 계획서

이 문서는 관리자가 서비스 안에서 결재 문서 양식을 직접 만들고 수정할 수 있게 하기 위한 투두리스트다. 1차 목표는 드래그 기반 편집기가 아니라, 필드 기반 양식 빌더와 미리보기, 기안 작성 자동 렌더링까지 완성하는 것이다.

## 진행 원칙

- [ ] 한 번에 하나의 작은 기능만 구현한다.
- [ ] 기존 `DocumentTemplate.schema`와 호환되는 방식으로 확장한다.
- [ ] 양식 변경이 기존 결재 문서를 깨뜨리지 않게 한다.
- [ ] 관리자 화면, 기안 작성 화면, 문서 상세, PDF 출력 흐름을 함께 확인한다.
- [ ] 완료한 작업은 이 문서의 체크박스를 갱신한다.

문서양식 디자이너는 아래 묶음 순서로 완성한다. 각 묶음은 화면, 데이터, 검증 기준을 함께 끝낸 뒤 다음 묶음으로 넘어간다.

구현 메모:

- 새로 생성되는 양식과 seed 양식은 `version: 1`이 포함된 기본 schema를 사용한다.
- 기존 운영 데이터처럼 `version`이 없는 `{ fields: [...] }` 구조도 v1로 정규화해 호환한다.
- 런타임에서 잘못된 schema를 만나면 기본 기안 구조로 fallback한다. 저장 화면에서는 strict validation 실패를 사용자에게 보여주는 방향으로 연결한다.

현재 구조:

- 관리자 문서양식 화면은 현재 양식명, 설명, 활성 여부만 수정한다.

## 2. Schema v1 정의

- [x] 필드 기본 구조 정의: `name`, `label`, `type`, `required`
- [x] 보조 속성 정의: `placeholder`, `helpText`, `defaultValue`
- [x] 지원 필드 타입 정하기: 한 줄 입력, 긴본문, 숫자, 날짜, 선택, 체크박스, 첨부파일
- [x] 선택형 필드 옵션 구조 정의
- [x] 필드 이름 중복 방지 규칙 정의
- [x] 필드 삭제 후 기존 문서 표시 정책 정하기
- [x] schema 버전 필드 추가 여부 결정

완료 기준:

- [x] schema 예시 JSON을 문서화한다.
- [x] 잘못된 schema를 검증할 수 있다.

Schema v1 예시:

```json
{
  "version": 1,
  "fields": [
    {
      "name": "title",
      "label": "제목",
      "type": "text",
      "required": true,
      "placeholder": "문서 제목을 입력하세요"
    },
    {
      "name": "content",
      "label": "기안 내용",
      "type": "textarea",
      "required": true,
      "helpText": "검토 배경, 요청 사항, 기대 효과를 입력합니다."
    },
    {
      "name": "approvalType",
      "label": "결재 구분",
      "type": "select",
      "required": true,
      "defaultValue": "normal",
      "options": [
        { "label": "일반", "value": "normal" },
        { "label": "긴급", "value": "urgent" }
      ]
    },
    {
      "name": "attachments",
      "label": "첨부파일",
      "type": "attachments",
      "required": false
    }
  ]
}
```

검증 기준:

- `version`은 `1`만 허용한다. 단, 기존 데이터 호환을 위해 `version`이 없으면 v1로 정규화한다.
- `fields`는 1개 이상 50개 이하 배열이어야 한다.
- `name`은 영문자로 시작하고 영문, 숫자, 밑줄만 허용하며 대소문자 구분 없이 중복될 수 없다.
- `type`은 `text`, `textarea`, `number`, `date`, `select`, `checkbox`, `attachments`만 허용한다.
- `select` 필드는 1개 이상의 `{ label, value }` 옵션이 필요하고 옵션 value는 중복될 수 없다.
- `defaultValue`는 필드 타입에 맞아야 하며 첨부파일 필드에는 사용할 수 없다.
- 나중에 구조화 입력값을 저장할 때는 문서 저장 시점의 field snapshot을 함께 남기고, 현재 schema에서 삭제된 필드는 기존 문서 상세와 PDF에서 snapshot label로 계속 표시한다.
