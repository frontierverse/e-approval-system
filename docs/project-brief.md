# 프로젝트 기준

## 제품

- 제품명: 결재온
- 성격: 사내전자결재 웹서비스
- 제공 형태: 웹 전용
- 모바일앱: 만들지 않는다.
- 모바일 대응: 별도 앱 없이 웹 화면이 작은 화면에서도 깨지지 않는 수준의 반응형만 고려한다.

## 사용자 역할

- 일반 사용자: 문서를 작성하고 결재 요청하며, 본인이 작성한 문서의 진행 상태를 확인한다.
- 결재자: 특정 문서의 결재선에 포함된 사용자다. 받은결재함에서 해당 문서를 승인하거나 반려한다.
- 관리자: 사용자, 부서, 직급, 문서 양식 같은 기준 정보를 관리한다.

## 문서 상태

- 임시저장: 작성 중인 문서다. 작성자만 수정할 수 있다.
- 결재 요청: 작성자가 결재를 요청한 상태다.
- 진행중: 결재선 중 한 명 이상이 처리했고, 아직 최종 완료되지 않은 상태다.
- 승인완료: 모든 결재자가 승인한 상태다.
- 반려: 결재자가 반려하여 결재가 중단된 상태다.
- 회수: 작성자가 결재 요청 후 결재 진행 전에 되돌린 상태다.

## 결재 액션

- 임시저장: 작성 중인 문서를 저장한다.
- 결재 요청: 결재선에 승인 요청을 보낸다.
- 승인: 현재 결재자가 문서를 승인한다.
- 반려: 현재 결재자가 문서를 반려한다.
- 회수: 작성자가 아직 처리되지 않은 결재 요청 문서를 되돌린다.

## MVP 문서 양식

첫 번째 문서 양식은 `일반 기안서`로 한다.

필드:

- 제목
- 기안 내용
- 결재선
- 첨부파일
- 작성자
- 작성일

첨부파일은 개발 환경에서는 로컬 저장소를, 운영 환경에서는 Vercel Blob 저장소를 사용한다.

## MVP 운영 규칙

- 결재는 지정된 순서대로만 진행된다.
- 작성자 본인은 결재자로 지정할 수 없다.
- 같은 문서 안에서 결재자는 중복 지정할 수 없다.
- 현재 순번의 결재자만 승인 또는 반려할 수 있다.
- 반려되면 이후 결재자는 처리할 수 없다.
- 승인완료 또는 반려된 문서는 수정할 수 없다.
- 작성자는 본인이 작성한 문서만 제출 문서함에서 볼 수 있다.
- 결재자는 본인이 결재선에 포함된 문서만 볼 수 있다.
- 관리자는 모든 기준 정보를 관리할 수 있지만, 문서 본문 수정 권한은 가지지 않는다.

## 문서 보관 정책

- 승인완료, 반려, 회수 문서는 처리일 기준 5년 동안 기본 보관한다.
- 5년이 지난 문서는 `보관 대상`으로 표시하고, 자동 삭제하지 않는다.
- 보관 대상 문서도 기존 조회 권한을 그대로 적용한다.
- 실제 삭제 또는 별도 외부 보관은 관리자 검토와 백업 정책이 정해진 뒤 진행한다.
