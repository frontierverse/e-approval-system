# 첨부파일 도장/서명 기능 개발 계획서

이 문서는 결재 문서 본문이 아니라 첨부파일 자체에 도장 또는 서명을 붙이는 기능을 만들기 위한 투두리스트다. 원본 첨부파일은 보존하고, 도장/서명이 합성된 서명본을 새 첨부파일로 생성하는 방향을 기준으로 한다.

## 기능 범위

- [x] 1차 범위는 `PDF`, `JPG`, `PNG` 첨부파일로 제한한다.
- [x] `hwp`, `docx`, `xlsx`는 1차 범위에서 제외하고 PDF 변환 후 날인이 필요하다고 안내한다.
- [ ] 법적 효력이 필요한 전자서명 인증서 연동은 별도 기능으로 분리한다.
- [ ] 도장/서명 기능의 용도는 내부 결재 표시와 감사 추적으로 정의한다.

## 기본 정책

- [x] 원본 첨부파일은 덮어쓰지 않는다.
- [x] 도장/서명본은 새 첨부파일로 저장한다.
- [x] 새 파일명 규칙을 정한다. 예: `원본파일명_서명본.pdf`
- [x] 원본 첨부파일과 서명본 첨부파일의 관계를 DB에 남긴다.
- [x] 누가, 언제, 어떤 첨부파일에 도장/서명을 붙였는지 감사 로그에 기록한다.
- [x] 다운로드 권한은 기존 첨부파일 권한 정책을 따른다.

## 도장/서명 이미지 관리

- [x] 사용자별 도장/서명 이미지 저장 필드를 DB에 추가한다.
- [x] `내 계정` 화면에 `결재 도장/서명` 관리 영역을 추가한다.
- [x] 도장/서명 이미지는 `png`, `jpg`, `webp`만 허용한다.
- [x] 투명 배경 PNG를 권장하되 일반 이미지도 업로드할 수 있게 한다.
- [x] 도장/서명 이미지 업로드, 교체, 삭제 서버 액션을 만든다.
- [x] 기존 프로필 이미지 업로드 로직과 저장소 정책을 재사용할 수 있는지 확인한다.

## 첨부파일 날인 UI

- [x] 첨부파일 미리보기 화면에 `도장 찍기` 버튼을 추가한다.
- [x] 버튼은 해당 문서를 결재할 권한이 있는 사용자에게만 보여준다.
- [x] 도장/서명 이미지가 등록되지 않은 사용자는 계정 설정으로 안내한다.
- [x] PDF 첨부파일은 페이지 선택 UI를 제공한다.
- [x] 이미지 첨부파일은 단일 화면에서 도장 위치를 지정하게 한다.
- [x] 사용자가 클릭 또는 드래그로 도장 위치를 잡을 수 있게 한다.
- [x] 도장을 여러 개 추가할 수 있게 한다.
- [x] 선택한 도장을 삭제할 수 있게 한다.
- [x] 도장 크기 조절 슬라이더를 제공한다.
- [x] 회전 기능은 1차에서 제외하거나 0도 고정으로 시작한다.

## 파일 합성 처리

- [x] 서버에서 PDF 위에 도장 이미지를 합성하는 라이브러리를 선정한다.
- [x] PDF 합성 후보로 `pdf-lib`를 검토한다.
- [x] 이미지 합성은 운영 환경의 native dependency 부담을 확인한 뒤 결정한다.
- [x] `sharp` 직접 의존성으로 JPG/PNG 합성까지 1차에 포함한다.
- [x] 합성 결과 파일의 MIME 타입과 파일 크기를 기존 첨부파일 정책에 맞춰 검증한다.
- [x] 합성 실패 시 원본 파일과 기존 첨부파일 목록이 변경되지 않게 한다.

## 결재 흐름 연동

- [ ] 승인 버튼 클릭 전 도장/서명본이 필요한 첨부파일이 있으면 안내한다.
- [x] 1차 정책은 현재 결재자가 결재 처리 전 또는 승인 처리 중에만 도장/서명할 수 있게 한다.
- [x] 이미 승인완료, 반려, 회수된 문서에서 추가 날인을 허용할지 정책을 정한다. 1차 정책은 허용하지 않는다.
- [x] 문서 상세에서 원본과 서명본을 구분해서 표시한다.
- [x] 서명본 첨부파일에는 `서명본` 배지를 표시한다.
- [x] 서명본 첨부파일은 잘못 생성했을 때 삭제할 수 있게 한다.
- [ ] 승인 의견과 별개로 첨부파일 서명 완료 상태를 확인할 수 있게 한다.

## 데이터 모델 후보

- [x] `User`에 도장/서명 이미지 저장소 필드를 추가한다.
- [x] `Attachment`에 원본 첨부파일 ID를 가리키는 필드를 추가한다.
- [x] `Attachment`에 서명본 여부 또는 첨부파일 종류를 나타내는 필드를 추가한다.
- [x] `Attachment`에 서명 처리자와 처리 시각을 저장할지 검토한다.
- [ ] `AuditAction`에 첨부파일 도장/서명 액션을 추가한다.

## 테스트

- [ ] 테스트 데이터에 도장/서명 이미지가 있는 사용자를 추가한다.
- [x] PDF 첨부파일에 도장 이미지가 합성되는 단위 테스트를 만든다.
- [ ] 권한 없는 사용자가 날인 액션을 호출하면 실패하는 테스트를 만든다.
- [ ] 원본 첨부파일이 변경되지 않는지 테스트한다.
- [ ] 서명본 첨부파일이 같은 문서에 연결되는지 테스트한다.
- [ ] 문서 상세에서 원본과 서명본 표시가 구분되는지 렌더링 테스트를 만든다.

## 1차 완료 기준

- [x] 사용자가 도장/서명 이미지를 등록할 수 있다.
- [x] PDF 첨부파일 미리보기에서 도장 위치를 선택할 수 있다.
- [x] 도장이 합성된 PDF가 새 첨부파일로 저장된다.
- [x] 문서 상세에서 원본과 서명본을 구분해 볼 수 있다.
- [x] 감사 로그에 첨부파일 도장/서명 처리 이력이 남는다.
